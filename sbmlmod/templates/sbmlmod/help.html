    Help

    SBMLmod is a web application that allows integration of data such as for example gene expression data, proteomics data or metabolomics data into an SBML model. On this help page you may find a step by step guide of how to use the application, as well as explanation of file formats.

    Step by step guide

    Step 1 – A. Input files

    Locate the files containing the model, the data to be integrated into the model, as well as a mapping file. The mapping file may be ommitted if the identifiers of the data file match exactly to the identifiers of the reactions id’s in the SBML model file. For more information on the files and options, scroll down to file formats.

    Example files are provided for testing the application. To use the example files, tick the check box labelled “Use example files” and proceed to Step 2. Details about the example files can be read here.

    You may choose to activate the “batch mode”. In this mode, the model will not only be modified depending on one data column, but on many. This will result in as many model files as data columns are given. All subsequent tasks will be performed on all models.

    Step 2 – B. Choose first task

    Depending on the files you uploaded in step 1, different tasks will be available. All tasks need a mathematical model in SBML format. This is also enough to run a steady state analysis. Depending on the data, you may also adjust local and global reaction parameters or set initial species concentrations. After performing the first task, other tasks may be chosen depending on the uploaded data files. Given you provide all files, you may for example first set the initial species concentration, adjust reaction parameters and finally run a steady state analysis on the resulting model file(s).

    Adjust reaction parameters

    In an SBML model, parameters are defined either globally or locally. Global parameters can be used by all reactions, while local parameters can only be used by the particular reactions it has been defined within. To adjust these parameters using a data file, follow these instructions:

    1. Decide whether you want to adjust the global and/or local parameters by ticking the check boxes in front of the Global and Kinetic law labels respectively.
    2. If you have selected Kinetic law (local parameters) you must also provide information about which parameter you want to adjust. This must be specified in the text field labelled Parameter to change. If you only want to change global parameters, the text field may be empty.
      • Example: If one of the parameters describes enzyme concentration, and you have named the parameter E_T, then write E_T into the text field in SBMLmod. Each enzyme in the model may have its own E_T parameter. SBMLmod will therefore attempt to find this parameter for each enzyme, and use the corresponding value from the data file to set the new value for E_T. Hence, it is important that you spelled the parameter name exactly the same for each enzyme, when you built the model.
    3. Both local and global parameters can be either replaced or scaled. Scaling the parameters means that the old parameter value will be multiplied by the respective value provided in the data file. This may be useful if the model contains for example liver specific activities for the enzymes, and then you have a data file with ratios of gene expression data from e.g. brain and liver. Replace means that the respective data value from the data file will be used to over-write the old value.
    4. Finally, a merge mode must be selected. Merge mode is used only, if two or more entries in the data file map to the same parameter in the model. If the data file contains gene expression data, there could be several genes that map to the same reaction, hence the value to represent the parameter must be chosen or calculated. The different merging possibilities are described here.
    5. You may start the task by clicking the “run task” button.

    Set initial species concentrations

    Initial species concentrations may be set according to some external data, for example growth medium conditions. The initial species concentrations will be set according to the data file. If the entries in the datafile don’t map directly to the entries in the model file, a mapping file is needed. No parameters are needed for this task.

    To start the task, click the “run task” button.

    Steady state analysis

    The uploaded or modified model(s) may be sent to a Copasi web service to run a steady state analysis. No parameters are needed for this task.

    To start the task, click the “run task” button.

    File formats

    Model (SBML) file

    The model must be provided in Systems Biology Markup Languange (SBML) format. This is an xml format provided by most, if not all, modelling tools. For further information, please see:

    Michael Hucka, Frank T. Bergmann, Stefan Hoops, Sarah M. Keating, Sven Sahle, James C. Schaff, Lucian P. Smith and Darren J. Wilkinson: The Systems Biology Markup Language (SBML): Language Specification for Level 3 Version 1 Core. Available from Nature Precedings (2010) DOI: 10.1038/npre.2010.4959.1

    Data file

    The data file should be a tab delimited text file. One header row and below one row for each data entry. The new model file(s) will be named according to the header file. If you don’t use the batch mode, you should also state the column to use for modifying the model. When using batch mode, each column will be used and multiple models will be produced, one for each data column. Have a look at the example files for a better understanding of the file format. The columns should contain the following information:

    Column 1:
    Data identifiers, e.g. gene identifiers in case of gene expression data, or metabolic compound identifiers in case of initial concentration data.
    Column 2 – n:
    Data values, e.g. one column from each sample in the case of gene expression data.

    Mapping file

    The mapping file should be a tab delimited text file with two or three columns. One row for each combination of SBML model reaction identifier and data identifier. If several data indentifiers map to the same reaction identifier, then these can be listed on separate rows. The different entries will be merged according to the Merge Mode selected. The mapping is case-sensitive! The columns should contain the following information:

    Column 1:
    SBML model identifiers, e.g. global parameter identifiers, reaction identifiers (for integration of local parameters), or species identifiers, depending on which method is used for the integration.
    Column 2:
    Data identifiers. These identifiers must match exactly the identifiers in Column 1 of the data file.
    Column 3 (optional):
    Key words COMPLEX and/or ISO as annotation for proteins that are either isoforms or part of a complex. This is column is only needed if “E-flux” is selected as Merge Mode.

    Merge modes

    Merge modes are only considered, when two or more entries in the data file match to one parameter.

    Max
    Of the data entries that maps to a particular parameter, the one with the highest value is chosen to scale or replace the parameter value.
    Min
    Of the data entries that maps to a particular parameter, the one with the lowest value is chosen to scale or replace the parameter value.
    Sum
    The sum of the data entries that maps to a particular parameter, is chosen to scale or replace the parameter value.
    Mean
    The mean of the data entries that maps to a particular parameter, is chosen to scale or replace the parameter value.
    Median
    The median of the data entries that maps to a particular parameter, is chosen to scale or replace the parameter value.
    E-flux
    Integrates the value as a function of the gene expression. Iso-enzymes may be expressed in different tissues. They act individually, and should be represented by separate reactions. In genome scale models, however, these are often represented by one reaction. During the integration this can thus be solved by summing up the gene expression values for iso-enzymes that map to the same reaction. For enzyme complexes one may assume that one of the subunits have a regulatory role, and hence the amount of active enzyme is related to the availablility of this regulatory subunit. E-Flux therefore uses the gene with the lowest expression value to represent the activity of the enzyme complex. To use this method an additional column must be present in the mapping file. The additional column should annotate the genes belonging to an enzyme complex or are iso-enzymes.
